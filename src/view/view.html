API REST Agenda

Utilize:
Metódo POST: /creat
Metódo GET: /read
Metódo GET: /read/id
Metódo PATCH: /update/id
Metódo DELETE: /delete/id

Em todos os casos erros inesperados serão retornados com o código 400 seguido do erro.

/creat:
Os dados a serem inseridos são {"nome":"string","telefone":"string","email":"string"}
Caso sejá inserido com sucesso será retornado código 200 {mensagem: 'Contato inserido com sucesso!'}
Caso o contato já exista será retornado código 400 {erro: 'Usuario já existe'}
Caso algum dos dados esteja faltando será retornado código 422 {erro: 'Nome e telefone são necessários'}
/read:
Os dados para cada contato a serem retornados serão {"_id": "string","nome":"string","telefone":"string","email":"string", "__v": 0}
Caso o não haja contato no banco será retornado código 422 {erro: 'Não foram encontrados registros'}
/read/id:
Os dados do contato serão {"_id": "string","nome":"string","telefone":"string","email":"string", "__v": 0}
Caso o contato não for encontrado no banco será retornado código 422 {erro: 'Contato não encontrado'}
/update/id
Caso o contato seja atualizado será retornado código 200 e {mensagem: 'Contato atualizado com sucesso!'}
Caso o contato NÃO seja encontrado será retornado código 422 e {mensagem: 'Contato não foi encontrado!'}
/delete/id
Caso o contato seja deletado será retornado código 200 e {mensagem: 'Contato excluido com sucesso!'}
Caso o contato NÃO seja encontrado será retornado código 422 e {mensagem: 'Contato não foi encontrado!'}
